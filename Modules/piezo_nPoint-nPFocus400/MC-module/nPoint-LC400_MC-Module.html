nPoint-LC400_MC-Module
The full LabView front panel of the
nPoint-LC400 MC module		 The part of the LabView front panel
which we typically show when it's
integrated in an interface

This software provides an interface for controlling the nPoint LC.400 controller for an attached 1-axis piezo stage.

It's the Module Conductor (MC) module and can be a part of the MC assembly, and receive commands from other MC modules in the assembly, including the Sequence Runner (SR). See
nPoint-LC400_MC-Module for the set of available commands and startup options. Typically, only the top row of controls are shown in the integrated GUI, with other values set up or determined automatically at startup. If other controls and indicators are desirable at the runtime, simply activate the vertical scrollbar in the MC config file.

The usage is straight-forward. Set the required position of the stage in the Ch1 Position control and the stage will move there. Ch1 Sensor displays the current position of the stage, updated every pos upd, ms .

The piezo stage, if present, is detected automatically, and the found stages are shown in the Found piezos indicator. At the moment, if the auto-connect, if only one piezo found is selected, the first stage is initialized automatically. Otherwise, in the current version of the code, an error is generated and the program halts. #fix

Upon initialization of the stage, Ch1 Range and Ch1 Position Units indicators will be filled.

There is no feedback from the stage controller indicating that the position has settles. And receiving a position feedback from the stage at high enough rate to analyze it for ringing (would require up to 2 KHz acquisition rate) isn't implemented and is potentially not-reliable. Thus, currently, in order for the stage to provide the feedback for the MC SR indicating that the move has concluded, user can set move dur, ms control to the manually determined settling time.

Logging

This MC module can log relevant messages via the MC asynchronous logger. For that, the logger has to be running concurrently, and either of the two following conditions be met. Either both the MC module and the logger are both LabView modules, and the name of the logger is the same as the dst field in the logger cluster, or the both of them have been started as part of an MC module assembly, and the logger is identified in the assembly by string in the dst field.

The resultant log file is a text file which can contain entries from multiple modules. The logger comes with a python parser which parses the log into a pandas DataFrame. However, in order to parse messages from particular modules they have to provide a parser for their messages (see the information on the MC logger for details). The parser for the messages produced by the nPoint-LC400 module can be found in MC-log-parser\piezo_nPoint_C400_log.py.
